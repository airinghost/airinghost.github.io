该站点始于2019年1月9日的《吃火锅记》。

1. 域名：Zhang 是姓氏拼音，Jet 是以前读培训班时从课本词汇表里挑的最短的英文名；最后选的 .COM 是因为它最常用。

2. 用过的服务：从 anyNode 到 Vultr 的 VPS，再从 GitHub 到 Netlify 到 Gitee 到 Coding 到 Vercel 的 Pages，然后从阿里的学生机到腾讯的轻量云、COS 和 CDN 再到华为云。之前用过 Coding.net 的“持续集成”功能来更新博客，但它几次改版后便没有再用。之后换用 GitHub Actions，非常方便。不论手机电脑，只要有个浏览器就可以打开 GitHub 记流水账了。后面也尝试过用手机或者电脑记事本写，写完后开 SSH 复制粘贴，但还是麻烦了些。

3. 网页生成与主题：与许多网友的选择一样，在最初的 VPS 阶段用的是 WordPress，因为当时面板都提供一键安装功能，主题选的是 Adams。然后换成了 Jekyll，因为不想再续费 VPS 了，喜欢上免费的 Pages 服务，而 GitHub 和 Gitee 默认就支持 Jekyll。当时有两款主题挺喜欢，分别叫 HardCandy 和 TMaize，可是配置文件看不懂，只能乱搞一通。之后投奔了 Hexo，主题仍与很多人一样选择了 NexT。但后来我觉着它顶部的导航栏每次都要点击下拉，这个操作在移动端上实在繁琐；又恰巧在 MoeCraft 的用户中心网站页脚看见了 Icarus 主题的链接，点进去瞧瞧发现很好看！于是果断换之，胡乱删改样式，用了两年。最后知道了 Hugo，确实快！而且只需一个可执行文件，没有厚重的 node_modules。刚开始准备用时，看中了一款叫 Stack 的主题，但在移动端上它也不能直接显示导航栏，也得点击下拉，而且模板看不懂、改不动，所以继续用 Hexo。之后想了想，反正 Icarus 也手动改过，与其老是删改缝补，不如根据改过的样式自己撸一个 Hugo 主题吧。说干就干，边敲键盘边看文档边调试，接着 Icarus-Lite 就这样出来了，也放在了 GitHub 上。目前回到纯手打 HTML 阶段。

4. LOGO：原本用的是我在手机备忘录随手描了几笔的笑脸，觉得很有趣就留了下来。现在直接简单地用了一个 💖

5. RSS：之前根据 @Xing 同学的建议，通过 Hugo 添加过 RSS 订阅，但现在因为是手打的，所以暂时没有了。
